## TODO [N] — New quiz type: "Which one is ___?" (image grid)

Add a new quiz type `image_match` to the lesson JSON schema and quiz engine. The user sees a 2x2 grid of images, each with a label underneath, and must tap the one that matches the prompt.

**JSON schema:**
```json
{
  "type": "image_match",
  "prompt": "Which one is 'araw'?",
  "audio": "audio/lesson_005/araw.mp3",
  "options": [
    { "image": "images/araw.jpg", "label": "araw", "correct": true },
    { "image": "images/buwan.jpg", "label": "buwan", "correct": false },
    { "image": "images/bituin.jpg", "label": "bituin", "correct": false },
    { "image": "images/ulap.jpg", "label": "ulap", "correct": false }
  ],
  "explanation": "Araw means sun."
}
```

**UI layout:**
- Prompt text at top: "Which one is 'araw'?" with speaker icon if audio exists.
- 2x2 grid of square-ish image cards below. Each card: rounded rectangle (12pt corners), the image filling the card, a label centred below the image.
- Tapping a card selects it (purple border highlight). Then the standard correct/incorrect feedback banner appears.
- **Image handling:** if the image file exists in the bundle, display it. If not, display the grey placeholder rectangle with the label text inside (same pattern as v0.3.3 teach card placeholders). This lets Ean build quiz content before sourcing all images.

**Quiz engine integration:**
- Add `image_match` to the `SlideType` enum (or equivalent).
- Shuffle option order on each presentation (same as other quiz types).
- Wrong answers cycle back into the queue per existing wrong-answer mechanic.

**Files to create:** `Views/Lesson/Slides/ImageMatchQuizView.swift`
**Files to modify:** Quiz engine / slide router (add new type), lesson data model (add `image_match` type + options schema).

## TODO [N+1] — New quiz type: sentence construction (word blocks)

Add a new quiz type `sentence_build` where the user constructs a translation by tapping word blocks in the correct order. This is the classic Duolingo "translate this sentence" format.

**JSON schema:**
```json
{
  "type": "sentence_build",
  "prompt": "Translate this sentence:",
  "source_text": "I already ate (polite).",
  "audio": "audio/lesson_002/kumain_na_po_ako.mp3",
  "correct_order": ["Kumain", "na", "po", "ako"],
  "distractors": ["ko", "ba", "siya"],
  "explanation": "'Kumain na po ako' is the natural word order."
}
```

**UI layout:**
- Top: prompt text + source text in a card/bubble (English sentence to translate).
- Middle: answer area. A horizontal row (wrapping to multiple lines if needed) where placed word blocks appear in order. Starts empty. Tapping a placed block removes it and returns it to the word bank.
- Bottom: word bank. All words from `correct_order` + `distractors`, shuffled, displayed as tappable pills. Tapping a pill moves it to the answer area and removes it from the bank.
- Below the word bank: "Check" button, enabled only when the answer area has at least as many words as `correct_order`.
- On check: compare the user's word sequence against `correct_order`. If correct, play correct chime + pronunciation of the full sentence. If wrong, show incorrect feedback, shake the answer area, and return all blocks to the word bank.

**Word pill behaviour:**
- Each pill plays `tap.wav` + light haptic on tap (from v0.3.2).
- Each pill plays its individual word pronunciation if the audio file exists (from v0.3.2 TODO N+3, Approach A).
- Placed pills in the answer area have a purple tint. Bank pills are light grey.

**Files to create:** `Views/Lesson/Slides/SentenceBuildQuizView.swift`
**Files to modify:** Quiz engine / slide router, lesson data model.

## TODO [N+3] — Wikimedia image pipeline

Set up a system for sourcing and bundling images from Wikimedia Commons for use in teach cards and image_match quizzes.

**This is a process, not a feature to code.** Create the following:

1. A `scripts/image_sources.json` manifest file that maps image keys to Wikimedia URLs and licence info:
```json
{
  "araw": {
    "source": "https://commons.wikimedia.org/wiki/File:Sun_in_February.jpg",
    "licence": "CC BY-SA 4.0",
    "author": "John Doe",
    "local_path": "images/araw.jpg"
  }
}
```

2. A `scripts/download_images.py` script that:
   - Reads `image_sources.json`
   - Downloads each image from Wikimedia (using the direct file URL, not the wiki page)
   - Resizes to a max dimension of 400px (keeps file size small for app bundle)
   - Saves to `Resources/Images/` with the key as filename
   - Generates an attribution file (`Resources/Images/ATTRIBUTIONS.md`) listing every image, its source, licence, and author

3. A `ATTRIBUTIONS.md` in the app bundle (required for CC BY-SA compliance). Link to this from the Profile/Settings screen as "Image Credits" or "Attributions."

**For Claude Code:** create the script scaffolding and the manifest format. Ean will populate the manifest with actual Wikimedia URLs as lessons are authored. The script handles the downloading and resizing.

**Files to create:** `scripts/image_sources.json` (empty template), `scripts/download_images.py`, `scripts/requirements.txt` (add `Pillow` for image resizing).
**Files to modify:** Settings/Profile view (add "Attributions" link).

## TODO [N+4] — Image placeholders throughout quizzes

Extend the grey placeholder pattern from v0.3.3 teach cards to all quiz types that reference images.

For any quiz view that tries to load an image from the bundle:
- If the file exists: display it, rounded corners (12pt), proper aspect ratio.
- If the file does not exist: display a grey rounded rectangle (#E5E5E5) at the expected size with a muted label showing the image key (e.g., "araw") centred inside in ~13pt grey text.

This applies to: `ImageMatchQuizView` (the 2x2 grid), `TeachCardView` (inline images), and any future view that loads lesson images.

Create a shared `LessonImageView` component:
```swift
struct LessonImageView: View {
    let imageName: String?
    var cornerRadius: CGFloat = 12
    var aspectRatio: CGFloat = 3/2
    
    var body: some View {
        if let name = imageName, UIImage(named: name) != nil {
            Image(name)
                .resizable()
                .aspectRatio(aspectRatio, contentMode: .fill)
                .clipShape(RoundedRectangle(cornerRadius: cornerRadius))
        } else if let name = imageName {
            // Placeholder
            RoundedRectangle(cornerRadius: cornerRadius)
                .fill(Color(.systemGray5))
                .aspectRatio(aspectRatio, contentMode: .fit)
                .overlay(
                    Text(name)
                        .font(.caption)
                        .foregroundStyle(.secondary)
                )
        }
        // If imageName is nil, render nothing
    }
}
```

Use `LessonImageView` everywhere instead of raw `Image()` calls for lesson content.

**Files to create:** `Views/Lesson/Components/LessonImageView.swift`
**Files to modify:** `TeachCardView`, `ImageMatchQuizView`, any other views that load lesson images.
