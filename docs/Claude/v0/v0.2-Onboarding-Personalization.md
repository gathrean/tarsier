## Full Onboarding Flow (v0.2)

```
1. Welcome screen          — mascot + "Learn Tagalog" + Get Started
2. Name                    — "What should we call you?" (first name only, text field)
3. Skill level             — "How much Tagalog do you know?" (3 cards)
4. Motivation              — "Why are you learning?" (pills, multi-select)
5. Emotional hook          — 1 tailored stat + sentence based on motivation
6. Daily goal              — "How much time per day?" (5 / 10 / 15 min)
7. Coach mark intro        — first teach card with animated hints
8. Session 1 (Po & Opo)    — full session, learn-by-doing
9. Session complete        — ring 1/5, Alam Mo Ba, +15 XP, "Magaling, [name]!"
10. Notification permission — "Want Tarsier to remind you to practice?" + time picker
11. Home screen             — roadmap with 1/5 progress on Po & Opo
```

## 1. User Name (Personalization)

**Screen:** "What should we call you?"
- Single text field, first name only. No last name, no email.
- Placeholder text: "Your first name"
- Skip option: small "Skip" link below. If skipped, default to no name (omit personalized strings).
- Store in SwiftData: `userName: String?`
- Position in flow: after Welcome, before Skill Level. User shares their name before sharing anything else. Sets a personal tone.

**Where the name appears:**
- Session complete: "Magaling, [name]!" (or just "Magaling!" if no name)
- App reopen: "Welcome back, [name]" on home screen (first launch of the day only, not every tab switch)
- Streak milestone: "5-day streak, [name]!"
- Lesson complete (5/5): "Ang galing mo, [name]!" (You're so good)
- AI Practice greeting: system prompt includes user name

**Rules:**
- Never use the name more than once per screen. It should feel warm, not robotic.
- If name is nil, all personalized strings fall back gracefully (just remove the name, don't show "null" or "learner").

## 5. Tappable Tagalog UI Words

**Scope:** Only Tagalog words used in the app's UI and feedback, NOT lesson content. Lesson vocabulary has its own teaching system.

**Words to make tappable:**

| Word | Context | Chip text |
|---|---|---|
| Tama! | Correct answer feedback | "Correct!" |
| Mali... | Wrong answer feedback | "Wrong / Incorrect" |
| Magaling! | Streak or perfect session | "Great job / Excellent!" |
| Alam Mo Ba? | Fun fact tooltip label | "Did you know?" |
| Tara! | Emotional hook CTA, future buttons | "Let's go!" |
| Sige | If used on any button | "Okay / Sure / Got it" |
| Mabuhay! | If used on welcome/achievement screens | "Welcome! / Cheers! (lit. 'long live')" |

**Visual treatment:**
- Dashed underline: functionalPurple (#6B5B9A) at 40% opacity, 1pt dash pattern
- On tap: floating chip appears ABOVE the word
- Chip style: functionalPurple background, white text, SF Rounded 13pt bold, 8pt corner radius, small downward-pointing triangle/caret connecting to the word
- Chip auto-dismisses after 3 seconds OR on tap anywhere else
- No animation on dismiss (just disappear), subtle scale-up animation on appear (0 → 1, 0.15s ease-out)

**First-time behaviour:**
- The FIRST time a user encounters each tappable UI word, the chip auto-shows for 1.5 seconds without requiring a tap. This teaches the user that these words are tappable.
- Track per-word in SwiftData: `seenUIWords: Set<String>`
- After auto-show, the dashed underline remains. User can tap again anytime to see the chip.

**Implementation:**
- Create a `TappableTagalogWord` SwiftUI view modifier or component
- Takes: word (String), translation (String)
- Renders: the word with dashed underline, handles tap gesture, shows/hides floating chip
- Use across all screens where Tagalog UI text appears

## 6. Review Prompt

**Trigger:** After completing first FULL lesson (all 5 sessions, progress ring fills to 5/5).

**Implementation:**
- Use `SKStoreReviewController.requestReview()` (Apple's native prompt)
- Apple limits this to 3 displays per 365 days per user. Don't build a custom modal.
- Track in SwiftData: `hasPromptedReview: Bool` (default false)
- Set to true after first trigger. Don't prompt again for this milestone.
- Future prompt triggers (v2.1+): after completing Chapter 2, after 7-day streak

**Timing in the flow:**
1. User completes Session 5 of Lesson 1
2. Lesson complete screen: confetti, XP, Alam Mo Ba, "Ang galing mo, [name]!"
3. User taps Continue
4. 0.5 second delay
5. SKStoreReviewController fires
6. User lands on home screen with completed lesson

**Do NOT:**
- Show a custom "Rate us!" modal before or instead of Apple's native prompt
- Ask "Are you enjoying Tarsier?" as a pre-gate (Apple discourages this)
- Prompt during onboarding Session 1 (too early, not enough value delivered)

## 7. Notification Permission

**Screen:** "Want Tarsier to remind you to practice?"

**When:** After Session 1 complete screen, before home screen.

**Design:**
- Mascot illustration (tarsier with a small clock or bell)
- "Pick a time that works for you"
- Time picker (scroll wheel), defaulting to 7:00 PM
- Two buttons: "Remind me" (primary, functionalPurple) and "Not now" (text link)
- "Remind me" triggers iOS notification permission dialog
- If user grants permission: schedule daily local notification at their chosen time
- If user denies or taps "Not now": skip, don't ask again during onboarding

**Notification content (rotate daily):**
- "Time to practice, [name]! Your streak is at [X] days."
- "Kumusta, [name]? Tarsier misses you."
- "5 minutes is all it takes. Tara!"
