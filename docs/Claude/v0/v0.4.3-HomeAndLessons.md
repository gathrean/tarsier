## TODO A — Home screen: chapter grid (replaces zigzag roadmap)

Remove the zigzag lesson path from the home screen entirely. Replace it with a
vertical scrolling list of **chapter nodes** in a staggered 2-column grid layout.
Tapping a chapter node navigates to a new **Chapter Detail** page (see TODO B).

Reference: HelloChinese home screen (Image 1 in the design reference).

### Chapter node layout

Use a `LazyVGrid` with two flexible columns. Chapters alternate positions in a
staggered pattern so the grid feels organic, not rigid:

- Row 1: 1 node centred
- Row 2: 2 nodes side by side
- Row 3: 1 node centred
- Row 4: 2 nodes side by side
- (repeat pattern, adapting to however many chapters exist)

If the chapter data specifies a `row_layout` of `"single"` or `"double"` in
`chapters.json`, respect that. Otherwise fall back to the alternating pattern.

### Chapter node appearance

Each node is a **circle (~100pt diameter)** containing an icon (SF Symbol or
emoji from the chapter's JSON data). Below the circle: the chapter title in
~15pt SF Rounded medium.

**Per-chapter accent colours.** Each chapter in `chapters.json` should have an
`accent_color` hex string. The node's circle fill uses that colour. This makes
each chapter feel distinct on the home screen. Suggested starting palette:

| Chapter | Colour name | Hex |
|---------|------------|-----|
| 0 — Introduction | Teal | #2AA198 |
| 1 — Magalang | Ube Purple | #5B48E0 |
| 2 — Taglish | Coral | #FF6B6B |
| 3 — Kain! | Mango | #F5A100 |
| 4 — Pamilya | Rose | #E8577D |
| 5 — Lakad | Sky Blue | #38BDF8 |
| 6 — Bilhin | Emerald | #34D399 |
| 7 — Bahay | Warm Brown | #C47A2B |
| 8 — Pakiramdam | Lavender | #A78BFA |
| 9 — Kwento | Deep Teal | #0D9488 |
| 10 — Salita | Indigo | #6366F1 |

These are suggestions. Ean will confirm final colours. Store them in
`chapters.json` so they're data-driven, not hardcoded per chapter in Swift.

**Progress ring.** Each chapter node has a circular progress ring around it
(like HelloChinese). The ring fills proportionally based on completed lessons
vs total lessons in that chapter. Ring stroke: 4pt, using the chapter's accent
colour at full opacity for the filled portion and 15% opacity for the unfilled
track.

**Node states:**

- **Not started:** accent colour fill at 20% opacity, icon in accent colour.
  Progress ring empty.
- **In progress:** accent colour fill at full opacity, icon in white. Progress
  ring partially filled. Subtle pulse animation on the node (scale 1.0 to 1.02,
  looping, 2s duration).
- **Completed:** accent colour fill at full opacity, white checkmark overlay on
  the icon. Progress ring fully filled.
- **Locked:** grey fill (#E5E5E5), lock icon in medium grey. No progress ring.

### Top header: ube purple gradient

Above the chapter grid, add a header area with a vertical gradient from
`#5B48E0` (ube purple, top) fading to transparent/white (bottom). This header
contains the streak/XP/hearts bar (white text/icons against the purple
background). The gradient should be ~120pt tall and scroll with the content
(not pinned).

Reference: HelloChinese uses an emerald-to-white gradient at the top of its
home screen. Same treatment, ube purple.

**Remove "Kuya <User>" from the top left.** The greeting moves to a bottom
sheet (see TODO C). The top bar now shows only: streak (flame + count), XP
(gem + count), hearts (heart + count), right-aligned or evenly spaced.

### Scrolling behaviour

The gradient header + chapter grid scroll together as one `ScrollView`. As the
user scrolls down, the purple gradient scrolls away naturally. No sticky/pinned
header.

### Data source

Read from the existing `chapters.json`. Add these new fields per chapter if
they don't exist:

```json
{
  "id": "ch_00",
  "title": "Introduction",
  "subtitle": "Abakada",
  "icon": "textformat.abc",
  "accent_color": "#2AA198",
  "lessons": ["lesson_000_intro"]
}
```

**Files to create:** `Views/Home/ChapterGridView.swift`, `Views/Home/ChapterNodeView.swift`
**Files to modify:** Main home/Learn tab view (replace roadmap with grid), `chapters.json` (add accent_color, icon fields), chapter data model (add new Codable fields)

## TODO E — Lesson slides: progressive reveal animation

Currently, teach cards render all their content at once. Update the lesson slide
engine so content appears one element at a time with a fade + slide-up
animation.

### Behaviour

Within a single teach card, each content element (heading, body text, example
card, image) is hidden on initial render. Elements appear sequentially:

- First element fades in + slides up (~0.3s ease-out) immediately when the
  card appears.
- Subsequent elements appear when the user taps "Continue" / taps the screen /
  swipes. Each tap reveals the next element.
- When all elements on the current card are visible, the next tap/Continue
  advances to the next card in the session (existing behaviour).

### Implementation approach

Each teach card's content can be modelled as an ordered array of "blocks":

```swift
enum TeachBlock {
    case heading(String)
    case body(String)
    case example(tagalog: String, english: String)
    case image(String)
    case alamMoBa(String) // if inline
}
```

Track a `@State var visibleBlockCount: Int = 1`. On each tap, increment by 1.
Render blocks from index 0 to `visibleBlockCount - 1`. Each block uses a
`.transition(.opacity.combined(with: .move(edge: .bottom)))` with
`.animation(.easeOut(duration: 0.3))`.

The progress bar at the top should NOT advance between block reveals within the
same card. It only advances when moving to the next card.

### Continue button label

When there are still hidden blocks on the current card, the Continue button
says "Continue" (or shows a down-arrow icon). When all blocks are visible, the
button changes to "Next" to indicate it will advance to the next card.

**Files to modify:** `TeachCardView.swift` (or the parent slide engine view),
teach card data model (if blocks aren't already structured this way, refactor
the card content into an ordered block array)

## TODO F — Lesson slides: density and breathing room

Update teach card layout to show less text per card, with bigger type sizes and
more vertical space for images/illustrations.

### Layout changes

- **Heading** (the Tagalog word or concept): 32pt SF Rounded bold. This is the
  hero element of the card. Centre-aligned.
- **Body text** (explanation): 18pt SF Rounded regular. Left-aligned. Max 2
  sentences per body block. If the current content has longer paragraphs, Ean
  will split them across multiple cards during content QA.
- **Example cards** (Tagalog + English pairs): displayed as distinct cards with
  generous padding (16pt all sides). Tagalog in purple (#5B48E0), 20pt bold.
  English below in grey, 16pt regular. Light purple background (#5B48E0 at 6%
  opacity), 12pt corners.
- **Image area:** if an image or SVG is present, it gets **at least 40% of the
  card's vertical space** (minimum ~240pt on a standard iPhone). Images sit
  above the text content, not squeezed between paragraphs.
- **Vertical spacing** between elements: 16pt (was ~20pt+ with uneven gaps).
  Consistent, not variable.
- **Horizontal padding:** 24pt on each side (gives the content room to breathe
  without feeling cramped).

### Content principle (for Ean, not Claude Code)

One idea per card. If a card currently has a heading + 3 sentences of body +
an example + an Alam Mo Ba, that's too much. Split it. The progressive reveal
(TODO E) helps, but the root fix is fewer words per card. Ean will do this
during content QA passes.

**Files to modify:** `TeachCardView.swift`, any shared layout constants file

## TODO G — Alam Mo Ba: collapsible button

Move the Alam Mo Ba fact from an inline section on teach cards to a collapsible
button positioned below the progress bar during lessons.

### Behaviour

- When a teach card or quiz card has an `alam_mo_ba_inline` field in the JSON,
  a small tappable button appears below the progress bar (top of screen area,
  above the card content).
- Button label: "Alam Mo Ba?" in ~13pt SF Rounded medium, with a small "?"
  icon (SF Symbol `questionmark.circle`) to the left. Text colour: #F5A100
  (gold). Background: transparent.
- **Do NOT use a lightbulb icon.** Lightbulb implies "hint" or "premium tip"
  in the Duolingo mental model. The "?" circle is neutral and curiosity-driven.
- Tapping the button expands a small card below the progress bar (slides down,
  ~0.25s ease-out). The card has a light gold background (#F5A100 at 8%
  opacity), 12pt corners, and contains the Alam Mo Ba text in 14pt regular.
- Tapping again (or tapping elsewhere) collapses the card.
- The button only appears on cards that have `alam_mo_ba_inline` data. On cards
  without it, the space is empty (no orphaned button).

### Completion reward Alam Mo Ba (unchanged)

The random fun fact shown at lesson completion (from `completion_reward.alam_mo_ba`)
is separate from this change. That stays as a full-screen celebratory card at
the end of a lesson. This TODO only affects the inline Alam Mo Ba during
lessons.

**Files to modify:** Lesson slide engine (add the collapsible button above card
content), `TeachCardView` (remove inline Alam Mo Ba rendering), quiz card views
(add support for the collapsible button if they have `alam_mo_ba_inline`)
