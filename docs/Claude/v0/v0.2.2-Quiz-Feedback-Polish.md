# v0.2.2 — Quiz Feedback & UI Polish

**Date:** 2026-03-12
**Scope:** Lesson quiz feedback UX, progress bar, home screen header, word order quiz fixes

---

## Changes

### Quiz Feedback Banner (Duolingo-style)
- Replaced the old inline feedback section with a compact bottom banner (~120-140pt)
- **Correct:** light green background (`#E8F5E9`), green checkmark + "Tama!" header, green CONTINUE button
- **Wrong:** light red background (`#FDECEA`), red X + "Mali" header, red CONTINUE button
- Explanation text shown in 1-2 lines (14pt, dark text, `.lineLimit(2)`) below the header when available
- Word order wrong answers now show the Mali banner with "Correct: ..." instead of silently resetting
- Fill-in-blank wrong answers show the correct answer in the banner

### Progress Bar
- Increased height from 6pt to 10pt (thicker, more visible)
- Fill changed from solid purple to left-to-right gradient (`functionalPurple` → `brandPurple`)
- Heart icon + counter added to the right side of the progress bar in lesson top bar

### Home Screen Header
- Removed "Tarsier" text from center of header
- XP and hearts grouped together on the right
- Added scroll fade gradient (warmWhite → transparent, 24pt) below header so content visually vanishes underneath
- Removed box-shadow glow from header icons

### Word Order Quiz
- Fixed word slots shifting when placing words — uses hidden text anchor (longest word piece) for consistent slot widths
- Wrong answers now show Mali feedback banner instead of just shaking and resetting
- Removed silent retry behavior on wrong word order — matches multiple choice and fill-in-blank UX

### Zigzag Path (Home Roadmap)
- Lesson nodes now show topic emojis (32 mapped) instead of numbers/icons
- Locked nodes show greyed-out emoji (`grayscale(1.0)`, `opacity(0.4)`)
- Active node pulsing glow centered via `.background()` instead of ZStack sibling
- Popover card shows "Chapter X Unit Y" subtitle
- Practice node moved to left alignment
- Dashed path line opacity reduced to 0.3

### Theme
- Added `correctBannerBg` (`#E8F5E9`) and `wrongBannerBg` (`#FDECEA`) color tokens
- Added `pathActive`, `pathLocked`, `nodeShadow`, `lockedFill` color tokens
- `PrimaryButtonStyle` made internal (was private) for reuse in feedback banner
