## What was built

### SoundManager service (`Services/SoundManager.swift`)
- Singleton that preloads 6 UI sounds at init: `correct`, `incorrect`, `tap`, `page`, `progress`, `lesson_complete`
- Tries `.wav` first, then `.mp3` for each sound
- Falls back to iOS system sounds (`AudioServicesPlaySystemSound`) when custom files aren't bundled
- Uses `.ambient` audio category — won't interrupt background music
- Reads mute state from `UserDefaults("soundEnabled")`, matching the existing `@AppStorage` toggle in ProfileView
- Sound files sourced from Pixabay (Content License — free commercial use, no attribution required)

### Sound files (`Resources/Sounds/`)
All 6 `.mp3` files added to bundle via `project.yml` resources entry:
- `correct.mp3` — ascending chime on correct answer
- `incorrect.mp3` — soft low tone on wrong answer
- `tap.mp3` — subtle click on word order pill tap
- `page.mp3` — soft whoosh on onboarding page advance
- `progress.mp3` — sparkle ding on session complete
- `lesson_complete.mp3` — longer celebratory sound on lesson complete

### Correct answer feedback (`LessonView.swift`)
- Plays `correct.wav` via SoundManager
- Fires `.success` haptic
- After 0.3s delay, auto-plays pronunciation audio for the correct answer via AudioPlayerService
- Pronunciation stops if user taps "Continue" before it finishes

### Wrong answer feedback (`LessonView.swift`)
- Plays `incorrect.wav` via SoundManager
- Fires `.error` haptic
- Heart icon in lesson header shakes using `ShakeModifier` (spring animation)
- Heart count decrements after shake

### Word order quiz tap (`QuizSlideView.swift`)
- Plays `tap.wav` + light haptic on each word pill tap
- Attempts to play individual word pronunciation via AudioPlayerService (Approach A: only if audio file exists in bundle for that word)
- `audioBasePath` derived from the card's audio path

### Onboarding page transition (`OnboardingFlow.swift`)
- Plays `page.wav` + soft haptic on user-initiated forward advances
- Silent on auto-advance screens (5, 7, 9) — `advanceOnce()` calls `advance(playSound: false)`

### Session/lesson complete (`SessionCompleteView.swift`)
- Plays `progress.wav` on session complete, `lesson_complete.wav` on final lesson session
- Timed to fire when ring animation reaches 100% (~0.9s after appear)
- Fires `.success` haptic

### Sound toggle (ProfileView — already existed)
- `@AppStorage("soundEnabled")` toggle was already in ProfileView
- SoundManager reads from the same UserDefaults key
- Toggle controls UI sounds only — pronunciation audio (AudioPlayerService) is unaffected

### Fill-in-blank keyboard fix (`QuizSlideView.swift`)
- Added `@FocusState` to auto-focus the TextField when fill-in-blank cards appear
- 0.3s delay before focus to let view settle
