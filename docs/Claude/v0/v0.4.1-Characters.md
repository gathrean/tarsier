# Tarsier v0.4.1 — Character System

> Continues from v0.4 (Features).
> Drop into `docs/Claude/v0/`.
> Introduces a cast of tarsier characters used during lessons and quizzes.

---

## Context for Claude Code

**Every character name is a Tagalog vocabulary word.** The characters passively teach family and social vocabulary just by appearing. This is intentional and should be referenced in the UI (e.g., "Lola" always has "(grandmother)" as a subtitle the first few times she appears, then just her name after the user has learned the word).

**Art approach:** Ean will draw all characters as head-only illustrations, same base Bunso head shape with small visual modifiers per character. For now, use emoji placeholders everywhere. Ean swaps in real art later.

**Speech bubble pattern:** Characters appear as a small head (~48pt) with a speech bubble next to them. The bubble contains the quiz prompt or example sentence. This replaces plain text prompts in quiz views.

## TODO [N+1] — CharacterBubbleView component

Create a reusable component that shows a character head + speech bubble. Used across all quiz types.

```
┌──────────────────────────────┐
│  [👵]  ╭─────────────────╮   │
│  Lola  │ Kumain na po    │   │
│        │ ako.            │   │
│        ╰─────────────────╯   │
└──────────────────────────────┘
```

**Layout:**
- Character head: 48pt circle. If the character has a real image asset (`character_lola` in asset catalogue), display it clipped to a circle. If not, display the emoji at ~28pt centred in a light grey circle.
- Character name below the head in ~11pt muted text. On the first 3 appearances of a character in the user's lesson history, also show the meaning in parentheses: "Lola (grandmother)". After that, just the name. Track appearance count in UserDefaults or SwiftData (`characterAppearanceCounts: [String: Int]` on UserProfile).
- Speech bubble: right of the character head. Rounded rectangle (12pt corners) with a small triangle tail pointing left toward the character. Light grey background (#F5F5F5) or white with a thin border.
- Text inside the bubble: the prompt or example sentence. Tagalog text in bold, English translation (if present) below in regular grey. Speaker icon inline if audio exists.

**Props:**
```swift
struct CharacterBubbleView: View {
    let character: TarsierCharacter
    let text: String
    var translation: String? = nil
    var audio: String? = nil
    var showMeaning: Bool = true  // caller decides based on appearance count
}
```

**Files to create:** `Views/Lesson/Components/CharacterBubbleView.swift`

## TODO [N+2] — Add character field to quiz JSON schema

Add an optional `character: String?` field to all quiz card types in the lesson JSON schema. When present, it references a `TarsierCharacter` raw value.

```json
{
  "type": "sentence_build",
  "character": "lola",
  "prompt": "Translate what Lola is saying:",
  "source_text": "Kumain na po ako.",
  "audio": "audio/lesson_002/kumain_na_po_ako.mp3",
  "correct_order": ["Kumain", "na", "po", "ako"],
  "distractors": ["ko", "ba"],
  "explanation": "Lola is politely saying she already ate."
}
```

When `character` is present on a quiz card:
- Replace the plain text prompt area with `CharacterBubbleView` showing the character saying the source text / prompt.
- The prompt text ("Translate what Lola is saying:") sits above the bubble as a smaller instruction label.

When `character` is nil:
- Render the quiz exactly as before. No character, no bubble. Plain text prompt. This preserves backwards compatibility with all existing lesson JSON.

Apply this to all quiz types: `multiple_choice`, `fill_blank`, `word_order`, `sentence_build`, `image_match`, and any future types.

**Files to modify:** All quiz slide views (add conditional CharacterBubbleView rendering), lesson data models (add optional `character` field to quiz card structs).

## Lesson authoring notes for Ean

When writing lesson JSON going forward:
- Assign characters to quiz cards that involve dialogue or sentence examples. "Translate what Lola is saying" is more engaging than "Translate this sentence."
- Match character to context. Lola and Lolo for polite/formal speech (po/opo). Pare and Mare for casual/Taglish. Ate and Kuya for everyday family conversation. Tatay and Nanay for household topics.
- Vary characters within a lesson. Don't use the same character for every card. A lesson about greetings might have Bunso meeting Lola (formal), then Pare (casual), showing register differences.
- Jowa appears latest (Chapter 6+) because the vocabulary and scenarios are more advanced (expressing feelings, casual speech, Taglish).
- Character intro cards go at the start of the first lesson in the chapter where that character is introduced.
