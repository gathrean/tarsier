## Part A — Bunso Onboarding Flow

The onboarding is a linear, screen-by-screen flow. Each screen advances on tap or button press. No back navigation (intentional — keeps momentum forward). Bunso appears on most screens as a companion illustration. The flow ends by launching the user's first lesson automatically.

### Screen sequence

1. **Splash/mascot intro** — Bunso waving. "Hi there! I'm Bunso!"
2. **Quick questions intro** — "Just a few quick questions before we start your first lesson!" (Bunso curious/excited pose)
3. **Language picker** — "What would you like to learn?" Header: "For English Speakers". Single option for now: Tagalog with the Philippine flag (Ean provides flag vector). Tappable card, not a list. Leave architecture flexible for adding Cebuano/Ilocano later (array of language options, even if length is 1 at launch).
4. **Attribution survey** — "How did you learn about Tarsier?" Single-select list: App Store, TikTok, Instagram, YouTube, Friend or Family, Other. Store as a string enum. This is analytics-only — no branching logic depends on it.
5. **Proficiency self-assessment** — "How much Tagalog do you know?" Five tappable cards, one selected at a time:
   - "I'm new to Tagalog"
   - "I know some common words"
   - "I can have basic conversations"
   - "I can talk about various topics"
   - "I can discuss most topics in detail"
   Store as an integer (0–4). For v1, all proficiency levels start at Lesson 1. The value is stored for future adaptive pathing and analytics only.
6. **Proficiency response** — Bunso responds based on selection. Examples: level 0 → "No worries, we'll start from the very beginning!"; level 1–2 → "Okay, we'll build on what you know!"; level 3–4 → "Nice! Let's sharpen those skills." Brief screen, auto-advances after ~1.5s or on tap.
7. **Name + title collection** — "What should I call you?" Text field for name (placeholder: "Your name"). Below it, a segmented picker or horizontal pill selector: Ate / Kuya / (neither). Include a small muted caption: "In Filipino culture, Ate and Kuya are terms of respect and closeness — like older sister and older brother." Store both values (name: String, preferredTitle: String? where nil = neither). These get sprinkled throughout the app: "Welcome back, Kuya Ean!", "Good evening, Ate Maria!", or just "Welcome back, Ean!" if neither is chosen.
8. **Motivation picker** — "Why are you learning Tagalog?" Multi-select (user can pick more than one). Options:
   - "Just for fun"
   - "Prepare for travel"
   - "Connect with people"
   - "Impress my partner"
   - "Impress other Filipinos"
   - "Connect with my family better"
   - "Understand Filipino media/music"
   - "For my career"
   - "Other"
   Store as `[String]`. Minimum 1 selection required to proceed.
9. **Motivation response** — Bunso reacts to the selected motivations. Show a brief animated response. Priority logic for which reaction to show (pick the first match):
   - If "Connect with my family better" is selected → Bunso with heart eyes: "That's the best reason. Let's make it happen!"
   - If "Impress my partner" is selected → Bunso blushing: "Kilig! They're going to love this."
   - If "Impress other Filipinos" is selected → Bunso flexing: "They won't know what hit them."
   - If "Just for fun" is selected → Bunso excited/jumping: "The best way to learn! Let's go!"
   - If "Connect with people" is selected → Bunso waving: "Filipinos are the friendliest. You'll fit right in."
   - Default (anything else / multiple without priority match) → Bunso thumbs up: "Great reasons. Let's get started!"
   Auto-advances after ~2s or on tap.
10. **Daily goal intro** — "Let's set up a learning routine!" (Bunso with a small calendar/clock prop)
11. **Daily goal picker** — "What's your daily learning goal?" Four tappable cards, one selected:
    - 5 min/day — "Casual"
    - 10 min/day — "Regular"
    - 15 min/day — "Serious"
    - 20 min/day — "Intense"
    Store as integer (minutes). Default highlight on "Regular" (10 min).
12. **Goal confirmation** — "That's about [X] new words in your first week!" Estimated word counts: 5 min → ~15 words, 10 min → ~30 words, 15 min → ~50 words, 20 min → ~70 words. Bunso celebrating.
13. **Three-month vision** — "Here's what you can achieve in 3 months." Three benefit cards stacked vertically, each with an icon and short description:
    - "Converse with confidence — Stress-free speaking and listening exercises"
    - "Build up your vocabulary — Common words and practical phrases"
    - "Develop a learning habit — Smart reminders, fun challenges, and more"
    Continue button at bottom.
14. **Notification pitch** — "I'll remind you to practice so it becomes a habit!" Bunso tapping wrist (watch-checking pose). After ~3 seconds on screen, trigger `UNUserNotificationCenter.requestAuthorization(options: [.alert, .badge, .sound])`. If the user denies, continue anyway — do not gate progress on notification permission. Show a continue button regardless of permission result.
15. **Widget pitch (v2 placeholder)** — "I'll cheer you on from your home screen!" Brief screen showing a mock widget preview (static image placeholder). Button text: "Okay! You can add my widget any time." This screen is informational only — no widget setup happens here. Mark this screen with a `// v2: implement actual widget prompt` comment.
16. **Subscription gate** — "How do you want to get started?" Two tappable cards:
    - Card 1 (RECOMMENDED badge, top): "Super Tarsier" — "Faster progress, no ads." Highlighted with purple border/fill.
    - Card 2 (below): "Learn for free" — "Core learning features, with ads."
    If "Super Tarsier" is tapped → show a brief interstitial "You'll learn faster with Super Tarsier!" then trigger the Superwall paywall event (`Superwall.shared.register(event: "onboarding_paywall")`). Superwall handles paywall display, pricing, and dismiss. RevenueCat handles the transaction.
    If "Learn for free" is tapped → proceed directly to loading screen.
    If user subscribes OR dismisses paywall → proceed to loading screen.
17. **Loading screen** — Bunso reading a book (Ean provides illustration). Two lines of animated loading text:
    - "Building your course..."

    Use a simple progress animation (indeterminate or timed ~2–3 seconds). This screen masks any actual data setup (persisting onboarding answers, preparing lesson state). After loading completes, transition directly into Session 1 of the first lesson (Po & Opo, per existing lesson architecture).

func personalGreeting(for profile: OnboardingProfile) -> String {

## Part B — Google Cloud TTS Pronunciation Audio

### Strategy

Pre-generate all pronunciation audio during authoring time. No runtime TTS API calls. Audio files ship as bundled app assets, keeping the app fully offline-capable and zero ongoing cost.

### Workflow

1. A Python script reads all lesson JSON files, extracts every unique Tagalog string that needs pronunciation (vocabulary words, example phrases, quiz prompts).
2. For each string, the script calls Google Cloud TTS API with language code `fil-PH` and a WaveNet voice (preferred) or Standard voice (fallback).
3. The returned audio is saved as `.mp3` to a structured directory: `Resources/Audio/{lesson_id}/{sanitised_tagalog_string}.mp3`.
4. The lesson JSON schema gets an optional `audio` field on teach cards and quiz cards pointing to the relative audio path.
5. Ean QAs each audio file. If a word sounds off, options: adjust SSML (speed, emphasis), try a different voice, or record a manual replacement.
6. Final QA'd audio files are added to the Xcode project as bundled resources.

## TODO [N+7] — TTS generation script

Create a standalone Python script at `scripts/generate_audio.py`. This script is run locally on Ean's machine, not in the app.

**Requirements:**
- `google-cloud-texttospeech` pip package
- Google Cloud service account credentials JSON (Ean sets up the project and downloads the key)
- Input: path to the lesson JSON directory
- Output: audio files in `Resources/Audio/`

**Script behaviour:**

```
Usage: python scripts/generate_audio.py --lessons path/to/lessons --output path/to/audio --credentials path/to/credentials.json
```

1. Walk the lessons directory, parse each JSON file.
2. Extract all Tagalog text from:
   - `teach` cards: the `tagalog` field (or whichever field holds the Tagalog string)
   - `quiz` cards: correct answer options, prompt text if in Tagalog
   - Any `example_sentence` fields
3. Deduplicate strings. Normalise whitespace and casing.
4. For each unique string, check if the output audio file already exists (skip if so — supports incremental runs).
5. Call Google Cloud TTS:
   - `language_code`: "fil-PH"
   - `voice`: prefer WaveNet if available for fil-PH, otherwise Standard. Use a female voice as default (configurable via CLI flag).
   - `audio_config`: MP3 encoding, speaking rate 0.9 (slightly slower than default for learner clarity — configurable via CLI flag).
   - SSML wrapping optional: for multi-word phrases, add a brief `<break time="200ms"/>` between sentence segments if needed.
6. Save each audio file as `{lesson_id}/{sanitised_string}.mp3`. Sanitise by replacing spaces with underscores, removing special characters, and truncating to 80 chars max.
7. Generate a manifest JSON (`audio_manifest.json`) mapping each Tagalog string to its audio file path. This manifest is used by the next TODO to wire audio into lesson JSON.
8. Print a summary: total strings processed, new files generated, skipped (already existed), errors.

**Files to create:** `scripts/generate_audio.py`, `scripts/requirements.txt` (just `google-cloud-texttospeech`)
