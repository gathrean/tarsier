Each option is a tappable pill: unselected = light grey fill + dark text, selected = purple fill + white text. Pills wrap to the next line when the row is full. This fits all 9 options on one screen without scrolling on most devices.

Keep multi-select behaviour (pick more than one). Keep minimum 1 selection to enable Continue.

## TODO [N+2] — Teach card density improvement

Current teach cards have too much whitespace and small text. Improve density without creating a wall of text. Reference: Duolingo info slides (bold heading, larger body text, inline illustration cards, explanation at bottom — dense but intentional, not cramped).

Changes to `TeachCardView`:
- **Heading** (the Tagalog word or grammar concept): bump to ~28pt bold. This is the focal point.
- **Body text** (explanation): bump to ~17pt regular. Current size feels like fine print.
- **Example cards** (Tagalog + English pairs): display as distinct rounded-rectangle cards with the Tagalog string in purple (#5B48E0) and the English translation below in grey. Light purple tint background (#5B48E0 at 6% opacity), 12pt corner radius.
- **Alam Mo Ba? callout**: keep at bottom, style as a distinct callout with light gold background (#F5A100 at 10% opacity) and a small lightbulb or star icon. Should feel like a bonus, not core content.
- **Reduce vertical spacing** between elements from ~20pt+ to ~12pt. Elements currently float in isolation.
- **Speaker icon**: if the card has an `audio` field, the speaker icon should sit inline next to the Tagalog heading, not below or separate from it.

**Files to modify:** `Views/Lesson/Slides/TeachCardView.swift`

## TODO [N+3] — Image placeholders in teach cards

Add an optional `image: String?` field to the teach card model in the lesson JSON schema. This is prep for adding Wikimedia images later.

When `image` is present and the file exists in the bundle, display it as a rounded rectangle (16pt corners, aspect ratio 3:2, full width minus horizontal padding) above the body text.

When `image` is present but the file is missing (or while Ean is adding images), display a grey placeholder rectangle at the same size with a muted "Image" label centred inside. This makes it visible where images will go.

When `image` is nil, render the card exactly as before. No placeholder, no empty space.

Update the TTS generation script's JSON parser to ignore the new field gracefully.

**Files to modify:** Teach card data model (add `image: String?`), `TeachCardView`, lesson JSON schema docs.

## TODO [N+6] — Analytics section in Profile

For v1, keep it simple. Two visual elements:

**Weekly activity chart:**
- A small bar chart showing minutes practiced per day for the last 7 days (Mon–Sun).
- Each bar uses the purple accent colour. Today's bar is highlighted or slightly brighter.
- Data source: track `practiceMinutes` per day in SwiftData (add a `DailyActivity` model with `date: Date` and `minutesPracticed: Int` if it doesn't exist). Write to it whenever a session completes.

- A simple dot grid showing the current month. Days with activity get a filled purple dot. Days without get an empty circle. Today is highlighted with a ring.
- Compact: fits in ~200pt height.

Both elements should feel lightweight. Not a full analytics dashboard. Just enough to make the user feel their progress is tracked.
