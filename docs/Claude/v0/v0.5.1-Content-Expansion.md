## What Changed

### Chapter-scoped lesson IDs

Lesson IDs are no longer globally sequential. Each chapter restarts numbering with a chapter prefix.

**Before:** `"lesson_id": "013"` (global counter, breaks when reordering chapters)
**After:** `"lesson_id": "ch02_001"` (chapter-scoped, self-contained)

All internal references follow the same pattern:
- Session IDs: `ch02_001_s1`
- Card IDs: `ch02_001_s1_c1`, `ch02_001_s1_q1`
- Character intros: `ch02_002_s1_intro_lola`
- Connection cards: `ch02_011_s1_conn_1`

### Curriculum reorder

Chapters reordered based on competitor analysis and heritage learner priorities. Family moved to Ch2 (emotional core), Food to Ch3 (first verbs). Old Taglish chapter dissolved, content distributed into Ch3 and later chapters. Old Salita chapter renamed to Roots.

| 2 | Family | Pamilya | ch02_pamilya |
| 3 | Food | Kain! | ch03_kain |

### New quiz type: image_match

A 2x2 or 4x4 matching grid. User matches left items to right items. Used in review and chapter review lessons.

```json
{
  "type": "quiz",
  "quiz_type": "image_match",
  "prompt": "Match each word to its meaning:",
  "pairs": [
    { "left": "Nanay", "right": "Mom" },
    { "left": "Tatay", "right": "Dad" },
    { "left": "Ate", "right": "Older sister" },
    { "left": "Kuya", "right": "Older brother" }
  ],
  "explanation": "Nanay = mom, Tatay = dad, Ate = older sister, Kuya = older brother."
}
```

If image_match quiz type rendering is not yet implemented from v0.4, implement it now. Display as a two-column matching exercise: left column shows the items to match, right column shows the answers in shuffled order. User taps one from each column to form a pair. Correct pairs lock in place. All pairs must be matched to proceed.
